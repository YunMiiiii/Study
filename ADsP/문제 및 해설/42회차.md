- 저장된 데이터 : 컴퓨터가 접근할 수 있는 저장 매체에 저장되어 있다.
- 공용 데이터 : 여러 사용자가 서로 다른 목적으로 데이터베이스의 데이터를 공동으로 이용할 수 있다.
- 변화하는 데이터 : 새로운 데이터의 삽입, 기존 데이터의 수정 및 삭제를 통해 항상 최신의 정확한 데이터 상태를 유지한다.

2. 데이터의 가치 패러다임 변화에서 디지털화된 정보들이 복잡한 관계를 형성함에 따라 이를 효과적으로 관리해야 하는 단계는?
<br>1. 디지털<br>2. 연결<br>3. 에이전시<br>4. 사물 인터넷
   <br><br>**답 : 3**
- 빅데이터 가치 패러다임의 올바른 시간 순서는 'Digitalization - Connection - Agency' 이며 각 특징은
- Digitalization (디지털화) : 사진, 동영상과 같이 아날로그 세상을 효과적으로 디저털화하는 것이 가치를 창출하는 시기로, 운영체제나 오피스 프로그램 등 다양한 컴퓨터 관련 소프트웨어들이 중요하던 시기
- Connection(연결) : 디지털화된 정보와 대상들을 서로 효과적이고 효율적으로 연결하는 것이 중요해진 시기다
- Agency (에이전시) : 사물인터넷(IoT)의 성숙과 함께 연결이 증가하고 복잡해지면서, 복잡한 연결을 효과적이고 믿을 만하게 관리하는 것이 중요해진 시기다.

10. 빅데이터로 인한 변화로 올바른 것은?
    <br>1. 사후조사 → 사전조사 <br>2. 양 → 질 <br>3. 전수조사 → 표본조사 <br>4. 인과관계 → 상관관계
 <br><br>**답 : 4**<br>
- 사전처리 → 사후처리 : 빅데이터 시대에는 데이터를 먼저 수집한 후 필요에 따라 분석하는 사후처리 방식으로 변화 발생
- 질 → 양 : 한 두개의 잘못된 데이터는 결과에 영향을 미치지 못하게 되어 빅데이터는 양이 많을 수록 더 정확한 분석이 가능
- 표본조사 → 전수조사 : IT 기술의 발달로 모집단 전체 조사가 불가능하던 과거와 달리 URL을 통한 설문조사와 같이 전수조사가 가능해졌다.
- 인과관계 → 상관관계 : 데이터의 양이 증가함에 따라 상관관계를 통해 특정 현상의 가능성을 포착할 수 있게 되어, 인사이트 도출 과정에서 상관관계의 중요성이 더욱 부각되고 있다.

12. 빅데이터의 4V 중 비즈니스 효과에 해당하는 것은?
    <br>1. Volume <br>2. Variety<br>3. Velocity <br>4. Value
    <br><br> **답 : 4**<br>
- Volume : 데이터 양의 증가
- Variety : 데이터 유형의 증가 (데이터 다양성)
- Velocity : 데이터 생성 속도(또는 처리 속도)의 증가
- Value : 데이터가 보유하고 있는 내적 가치
  <br>
- 이 중 비즈니스적 효과를 의미하는 것은 Value이며, 나머지 Volume, Variety, Velocity는 데이터 분석에 있어서 투자 비용 요소로 여겨진다.


13. 기업의 데이터 분석의 성숙도는 4가지 수준으로 나누어더 진단할 수 있다. 다음 중 분석을 진화시키고 혁신 및 성과 향상에 기여하는 단계는?
    <br>1. 도입 <br>2. 활용<br>3. 확산<br>4. 최적화
    <br><br>**답 : 4**<br>
- 일반적인 데이터 분석 성숙도 모델에 대한 이해를 바탕으로 각 단계는 다음과 같은 특성을 갖는다. 따라서 위 문제는 최적화에 대한 설명이다.
  - 도입 : 데이터 분석을 처음 시작하는 단계로, 환경 및 시스템 구축을 위한 기본적인 분석 도구와 방법을 도입하는 단계이다.
  - 활용 : 도입된 분석 도구와 방법을 실제 업무에 적용하고 분석 결과를 업무에 적용하는 단계이다.
  - 확산 : 데이터 분석의 가치를 인식하고 전사 차원에서 분석 관리 및 공유 수행하며 분석 문화와 방법을 확산 시키는 단계이다.
  - 최적화 : 데이터 분석을 통해 얻은 인사이트를 바탕으로 비즈니스 프로세스를 지속적으로 개선하고, 새로운 기회를 발굴하며, 혁신 및 성과 향상에 기여하는 단계이다.

20. 분석 과제를 관리할 때 고려 사항에 해당하지 않는 것은?
    <br>1. 데이터 분류<br>2. 데이터 규모<br>3. 분석 속도<br>4. 분석 정확도

<br><br>**답 : 1**<br>
- 분석 과제를 관리할 때 고려할 사항 5가지
  - 데이터 규모(양): 데이터 양이 방대할 경우 데이터 관리를 위한 다양한 기술적, 환경적 요소가 뒷받침되어야 하며 이는 프로젝트 비용과 직결된다.
  - 데이터 복잡도 : 분석의 대상이 되는 데이터 유형 역시 프로젝트 비용에 직결된다.
  - 분석 속도 : 결과를 얼마나 신속하게 도출할 수 있는지는 프로젝트 일정 및 분석 모델의 실용성과 연결된다.
  - 분석 복잡도 : 필요한 리소스, 시간, 전문성 수준 등을 결정하는 데 영향을 준다.
  - 분석 정확도 : 분석 결과의 품질과 신뢰성을 결정하는 요소
