9. 빅데이터의 가치 산정이 어려운 이유로 부적절한 것은?
   <br>1. 데이터 활용 방식<br>2. 새로운 가치 창출<br>3. 폐쇄적 특성<br>4. 분석기술 발전

<br><br>**답 : 3**<br>
- 데이터의 가치 산정이 어려운 이유 4가지
  - 데이터 활용 방식 : 빅데이터는 다목적으로 재사용되고 재조합하는 특성이 있어, 누가 언제 어디서 어떻게 활용했는지 정확하게 파악하기 어렵다. 
  - 새로운 가치 창출 : 빅데이터 분석을 통해 기존에 없던 새로운 가치가 창출될 수 있다. 이러한 새로운 가치는 기존의 방식으로 측정하기 어려워 가치 산정에 어려움을 준다.
  - 분석기술 발전 : 빅데이터 분석 기술이 계속 발전함에 따라 과거에는 가치가 없다고 여겨졌던 데이터도 새로운 가치를 가질 수 있게 됐다. 이러한 기술의 변화로 인해 데이터의 가치를 일정하게 평가하기 어려워진다.
- 폐쇄적 특성은 빅데이터의 특성과 맞지 않다. 빅데이터는 오히려 개방적이고 공유 가능한 특성을 가지고 있어, 다양한 분야에서 활용될 수 있다. 따라서 이는 빅데이터의 가치 산정이 어려운 이유로 부적절하다.

16. 분석 거버넌스의 구성요소가 아닌 것은?
    <br>1. 비용<br>2. 조직<br>2. 데이터<br>4. 시스템
    <br><br>**답: 1**
    <br>
- 분석 거버넌스의 구성요소 5가지
  - 조직(Organization) : 분석 기획 및 관리를 수행하는 조직
  - 과제 기획 및 운영 프로세스 (Process) : 분석 과제를 기획하고 운영하는 프로세스
  - 분석 관련 시스템 (System) : 분석에 필요한 IT 시스템과 인프라
  - 데이터 (Data) : 분석에 활용되는 데이터
  - 분석 관련 교육 및 마인드 육성 체계 (Human Resource) : 분석 역량 강화를 위한 교육과 조직 문화 조성

17. 단계별 분석 로드맵을 수립할 때 분석 유효성 검증 단계에서 수행해야하는 과제는 무엇인가?
    <br>1. 파일럿 테스트 <br>2. 분석 과제 정의<br>3. 유관 시스템 고도화 <br>4. 빅데이터 활용 시스템 구축
    <br><br>**답 : 1**<br>
- 단계별 분석 로드맵 수립 시 추친 과제
  - 분석 체계 도입 단계
    - 분석 기회 발굴 및 과제 정의
    - 분석 로드맵 수립
  - 분석 유효성 검증 단계
    - 알고리즘 및 아키텍처 설계
    - 분석 과제 Pilot 수행

    - 유관 시스템 고도화<br>

18. KDD 분석 방버론에서 이상값 처리를 수행해야하는 단계로 올바른 것은?
    <br>1. 데이터셋 선택<br>2. 데이터 전처리 <br>3. 데이터 변환 <br>4. 데이터 마이닝
    <br><br>**답 : 2**
    <br>
- KDD 분석 방법론의 데이터 전처리 단계에서는 다음과 같은 업무를 수행한다. 
  - 잡음, 이상치, 결측치를 파악하여 제거하거나 의미있는 데이터로 재가공
  - 추가로 요구되는 데이터셋이 있다면 데이터 선택 프로세스를 다시 실행
- 이상값 처리는 모든 분석방법론에 있어서 데이터 전처리 혹은 데이터 준비 단계에 속한다. 다만 KDD 분석방법론에서는 이상값 처리를 직접적으로 언급한다.

19. 하향식 접근의 문제 탐색을 위한 방법 중 하나로 유사업계 벤치마킹을 통한 분석 기회를 발굴하는 것은?
    <br>1. 외부 참조 모델<br>2. 거시적 관점의 분석 기회 발굴 <br>3. 시장 니즈 탐색<br>4. 역량 재해석
    <br><br>**답 : 1**
    <br>
- 하향식 접근법의 1단계인 문제 탐색 단계에서는 다음과 같은 방법들이 있다. 
  - 비즈니스 모델 탐색 기법
  - 분석 기회 발굴 범위의 확장
  - 외부 참조 모델 기반 문제 탐색 및 분석 유스케이스 정의
- 이 중에서 유사 업계 벤치마킹을 통한 분석 기회 발 문제 ㅌ당하는 것은 " 외부 참조 모델 기반 문제 탐색"이다.

20. 분석 조직에 대한 설명으로 부적절한 것은?
    <br>1. 전사적 관점의 분석 과제를 도출한다.<br>2. 타 부서간 유기적인 협력을 통해 업무를 수행한다.
    <br>3. 업무의 최종 의사결정을 수행한다.<br>4. 집중형 또는 분산형 구조의 형태를 이룬다.
    <br><br>**답 : 3**<br>
- 1 분석 조직의 역할 중 하나는 전사 및 부서의 분석 업무를 발굴하는 것이다.
- 2 분석 구조 고려사항 중 하나가 분석 전담조직과 타 부서 간 유기적인 협조와 지원이 원활한 구조인지를 고려해야 하는 것으로 분석 조직은 타 부서와의 유기적 협력이 필요하다.
- 3 분석 조직의 역할은 기업 내 존재하는 빅데이터 속에서 인사이트를 찾아 전파하고 이를 Action화하는 것으로 최종 의사결정은 일반적으로 경영진이나 해당 업무 책임자의 역할이다.
- 4 분석 조직의 유형으로 집중형, 기능 중심, 분산형 구조가 있다.
