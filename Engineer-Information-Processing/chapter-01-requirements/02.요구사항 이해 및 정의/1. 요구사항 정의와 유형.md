### 01. 요구사항 정의와 유형
- **요구사항**이란 소프트웨어가 해결해야 할 문제에 대한 **기능과 제약 조건을 정의한 것**으로
  사용자가 시스템에 기대하는 서비스, 품질, 조건 등을 포함한다.


**기능 요구사항**
- 시스템이 빈드시 수행해야 할 **기능 중심의 요구**
- ex) 로그인, 상품 검색, 게시글 작성

**비기능 요구사항**
- 품질, 성능, 보안 등 **동작 방식이나 제약 조건**
- ex) 1초 이내 응답, 보안성, 확장성

**사용자 요구사항**
- **사용자 관점**에서 표현된 요구사항
- 친숙하고 이해하기 쉬운 표현 사용
- ex) 업무 흐름, 인터페이스

**시스템 요구사항**
- **개발자 관점**에서 기술적으로 표현된 요구사항
- DB 구조, API 명세 등 상세한 기술 명세 포함
- 예: 테이블 구조, 연동 방식, 프로토콜
